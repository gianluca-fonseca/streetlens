---- Defects found and fixed in this unit ----
1. frameQuality.reason: strict json_schema cannot omit a key, so "nothing was
   wrong" arrives as reason:null, but the frozen captureFrameQualitySchema types
   reason as .optional() (undefined, not null). EVERY usable frame would have
   failed validation. Found by the mocked test on its first run.
2. Per-session budget could not fire on a session's last frame: the pre-check
   only blocks a frame from STARTING, so with no next frame to block a session
   could overrun its cap and still finish review_ready, overrun unsurfaced.
3. temperature: gpt-5 reasoning models 400 the entire request. Every production
   call would have failed. Found by the live smoke's first real call.
4. insufficient_quota (a 429) was retried 3x with backoff though it can never
   succeed. Now fails fast.

---- Blockers needing Conductor action ----
A. CRITICAL — the 0013 storage upload policy can never pass; nobody can upload a

   capture_frames under RLS with zero policies, so it matches nothing, ever.
   Fix committed as 0016 (NOT applied; I do not run DDL per the seed).
B. HIGH — OPENAI_API_KEY has no quota, so the real token counts, prompt-cache
